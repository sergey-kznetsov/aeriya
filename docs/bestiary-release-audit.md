# Релизный аудит бестиария Aeria Core

## Назначение

Этот документ фиксирует итоговый аудит бестиария перед переходом к финализации handout-карточек Теневого и Палящего Осколков.

Аудит разделяет четыре уровня:

1. основной source-based бестиарий;
2. расширенный source-based слой;
3. модульные существа, созданные для сцен и локаций;
4. технические cleanup-задачи перед Foundry-конвертацией.

## Дубли и разделение ролей

В бестиарии есть похожие существа, но они разведены по ролям и не являются дублями:

| Пара / группа | Решение |
|---|---|
| Дух / Паразитический Дух / Дух Молчаливого Зева | разные CR, масштаб и назначение: базовая нежить, паразитическая сцена, boss-use дух Озерника |
| Тень, адаптированная / Голодная тень / Семиглазая тень | разные регионы и функции: классическая низкая нежить, лесная/городская угроза отчаяния, модульная сцена Семиглазого |
| Болотный шатун / Болотная химера / Болотная гарпия / Глубинник | один биом, разные роли: brute, мутант, летающая приманка, разумный водный обитатель |
| Жаровой сторож / Огненный страж | модульная локальная угроза и source-based конструкт-страж |
| Песчаный бегун / Песчаная пиявка / Поющий бархан / Песчаный червь | один регион, разные уровни угрозы и масштаб встречи |
| Ледяной сторож / Ледяной плясун / Морозный исполин / Йормунга | разные уровни северной угрозы: локальный страж, фейская ловушка, крупный великан, финальный фрагмент |

Вывод: критичных дублей, требующих удаления, нет. Похожие существа оставлены как разные encounter-инструменты.

## Проверка запрещённых старых механик

В новых партиях 03–08 сознательно не используются механики:

- продажи / потери имени;
- потери тени;
- кристалла души;
- кражи памяти как необратимой механики;
- пепла как товара.

Опасности переписаны через стандартные игровые эффекты: урон, состояния, свет/темноту, страх, истощение, захват, оглушение, контроль поля, ритуальные последствия и сюжетные условия сцены.

## Технический cleanup перед Foundry Actor pack

Остался один известный технический хвост: `source-import-batch-01.md` содержательно считается final и включён в индексы как `actor-ready-final`, но его frontmatter всё ещё содержит старый статус `actor-ready-import`.

Требуемое действие: короткая техническая нормализация frontmatter этого файла до:

```yaml
source: "book-iii-and-vii-final"
status: "actor-ready-final"
```

Предыдущая попытка полной перезаписи файла была заблокирована коннектором, поэтому задачу нужно выполнить отдельным безопасным cleanup-проходом или локальным патчем.
