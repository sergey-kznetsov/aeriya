# Проверка карточек контента

Этот документ описывает проверку Markdown-карточек перед импортом в Foundry.

## Зачем нужна проверка

Контент Аэрии переносится постепенно: города, фракции, духи, НИПы, монстры, предметы, ритуалы, handouts и таблицы. Чтобы модуль оставался удобным во время игры, каждая карточка должна иметь одинаковую структуру.

Проверка нужна, чтобы убедиться, что у карточки есть служебные поля, тип контента указан корректно, а данные готовы к генерации машинного индекса.

## Обязательные поля

Каждая карточка должна начинаться с YAML-блока. Обязательные поля: type, name, shard, region, source, status.

## Допустимые типы

Допустимые типы карточек: city, faction, npc, monster, spirit, ritual, item, handout, region, rolltable, scene.

## Команда проверки

После установки репозитория локально можно выполнить команду node scripts/dev/validate-content-cards.mjs.

Если карточки корректны, скрипт выведет количество проверенных файлов. Если есть ошибка, он покажет путь к файлу и список проблем.

## Связь с индексом

Валидатор не создаёт индекс. Для создания машинного индекса используется скрипт node scripts/dev/build-content-index.mjs.

Правильный порядок работы: добавить или обновить Markdown-карточки, проверить их валидатором, сгенерировать content-index.json, затем подключить индекс к Foundry-интерфейсу модуля.

## Принцип

Markdown-карточки остаются источником правды. JSON-индекс нужен только для быстрой навигации модуля и не должен редактироваться вручную.
