# Smoke test Aeria Core в Foundry

## Цель

Быстро проверить, что модуль ставится, активируется и может загрузить текущий контент как Journal Entry.

## Подготовка

В локальной копии репозитория выполнить:

```bash
npm run foundry:check
```

Эта команда проверяет markdown-карточки, пересобирает индекс, создаёт Foundry Journal Entry export и сверяет структуру результата.

После этого в модуле должны появиться:

```text
build/foundry/journal-entries.json
build/foundry/aeriya-journals/_source/*.json
build/foundry/aeriya-journals/manifest.json
```

## Установка

Скопировать папку проекта в:

```text
FoundryVTT/Data/modules/aeriya
```

В чистом мире D&D 5e включить модуль `Aeria Core`.

## Проверка модуля

| Проверка | Ожидаемый результат |
|---|---|
| Модуль виден в списке модулей | `Aeria Core` отображается |
| Модуль включается | ошибок загрузки нет |
| Консоль | `game.aeriya` существует |
| Settings | настройки реакций, точных попаданий и ритуалов видны |
| Язык | русские строки отображаются корректно |

## Проверка импорта Journal Entry

Открыть Script Macro или консоль и выполнить содержимое файла:

```text
scripts/foundry/import-journal-entries.js
```

Ожидаемый результат:

| Проверка | Ожидаемый результат |
|---|---|
| Импорт запускается от GM | ошибки прав нет |
| JSON найден | `journal-entries.json` загружается |
| Папки созданы | Cities, Factions, Handouts, Regions и другие папки появились |
| Записи созданы | Journal Entry появились в мире |
| Повторный запуск | записи обновляются, а не дублируются |
| Source flag | у записей есть `flags.aeriya.sourcePath` |

## Что проверить вручную после импорта

Открыть минимум:

- один город Срединных Земель;
- один handout Срединных Земель;
- один материал Теневого Осколка;
- один материал Палящего Осколка;
- один набор встреч;
- один мини-сценарий.

## Если импорт не работает

Проверить:

1. Выполнен ли `npm run foundry:check` без ошибок.
2. Есть ли файл `build/foundry/journal-entries.json` внутри папки модуля.
3. Есть ли staging-папка `build/foundry/aeriya-journals/_source`.
4. Совпадает ли путь в `AERIA_JOURNAL_JSON_URL`.
5. Выполняется ли код от GM-пользователя.
6. Есть ли ошибки в консоли Foundry.

## Когда smoke test считается пройденным

Тест считается пройденным, если модуль активируется, импорт создаёт Journal Entry, повторный импорт обновляет существующие записи, а мастер может открыть handouts и мини-сценарии прямо в Foundry.
