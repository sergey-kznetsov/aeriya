# Прогресс переноса контента

Модуль Аэрии собирается с нуля как надстройка над D&D 5e.

Старые compendium packs не копируются. Новые документы Аэрии являются основным источником. Прежний мир используется только как точечный справочник для отдельных сущностей.

## Текущий статус

### Срединные Земли

Готов первый справочный слой городов и фракций. Добавлен быстрый игровой слой:

- 8 НИПов;
- 4 сцены;
- 3 таблицы;
- 2 поручения;
- 2 мини-сценария;
- player-facing handout;
- GM handout;
- отдельный игровой индекс Срединных Земель.

### Теневой Осколок

Готов базовый игровой слой по трём регионам:

- Пепельная Степь;
- Лес Чёрного Шёпота;
- Вечные Льды.

Добавлен быстрый слой для запуска за столом:

- 6 поручений;
- 3 мини-сценария;
- 3 GM handouts;
- 2 новых player-facing handouts плюс чайная памятка Пепельной Степи;
- общая таблица поворотов поручений;
- отдельный игровой индекс Теневого Осколка.

Окрам остаётся отдельной точкой сверки со старым миром `shadow-lands`, но это не блокирует сборку первого рабочего модуля.

### Палящий Осколок

Готов самый полный игровой слой:

- региональные карточки;
- 15 городов и локаций;
- фракции;
- НИПы;
- предметы;
- сцены;
- существа;
- духи;
- ритуалы;
- таблицы;
- поручения;
- памятки;
- мини-сценарии.

Следующее: не расширять без необходимости, а переводить контент в Foundry packs.

## Технический статус

Уже есть:

- `module.json`;
- JS-слой модуля;
- русская локализация;
- стили;
- валидатор карточек;
- сборщик Journal Entry JSON;
- структура `assets/`;
- asset manifest.

Не хватает до хорошего модуля:

- реальные compendium packs;
- обновление `module.json` с packs;
- release zip;
- тест в чистом мире Foundry;
- первые Actor/Item/RollTable/Scene документы вместо markdown-заготовок.

## Ближайший приоритет

1. Добавить Foundry pack/export план.
2. Сделать первый `aeriya-journals` pack или максимально близкий к нему экспорт.
3. После стабильного Journal Entry pack переходить к картинкам, картам, портретам и токенам.
