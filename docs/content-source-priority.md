# Приоритет источников при переносе контента

Этот документ фиксирует правила переноса материалов в модуль **Aeria Core**.

Модуль собирается с нуля для установки в новый чистый мир Foundry на системе D&D 5e. Старый мир `shadow-lands` не переносится как готовый мир и не используется как готовый набор compendium packs.

## Основной принцип

При переносе материалов в модуль приоритет имеют новые документы проекта Аэрии.

Новые документы считаются основным источником для:

```text
регионов;
городов и поселений;
фракций;
духов;
ритуалов;
NPC;
монстров;
предметов;
бестиариев;
локаций;
книг и handouts;
правил песочницы;
боевых механик;
некромантии;
точных попаданий;
реакций;
эпических действий.
```

Если новая документация и старые материалы расходятся, по умолчанию используется новая документация.

## Старый мир shadow-lands

Архив `shadow-lands.zip` рассматривается только как донорский и справочный материал.

Из него нельзя автоматически переносить:

```text
готовый world.json;
старые compendium packs целиком;
пользователей;
сообщения чата;
fog of war;
настройки мира;
боевые сцены;
плейлисты и аудио;
внешний контент;
служебные файлы LevelDB;
материалы, не связанные напрямую с миром Аэрии.
```

Старый мир можно использовать точечно, если в нём какая-то часть мира описана точнее, чем в новых документах.

## Исключение: точные старые интерпретации

Некоторые материалы старого мира могут быть более точными или более удачными, чем текущие новые версии. В таких случаях их можно использовать как приоритетный источник, но только точечно и после проверки на соответствие актуальному лору.

Пример:

```text
Город Окрам — при переносе города Окрам следует ориентироваться на старый мир shadow-lands, если там дана более точная интерпретация города.
```

Это правило не означает, что старый мир переносится целиком. Оно означает только то, что отдельные сущности можно восстановить по старой версии, если она лучше соответствует замыслу.

## Что считается связанным с миром Аэрии

В модуль переносится только то, что напрямую относится к Аэрии:

```text
авторские города;
авторские фракции;
авторские NPC;
авторские монстры;
авторские предметы;
авторские ритуалы;
авторские духи;
региональные правила;
сюжетные события;
внутриигровые книги;
локации;
сцены без привязки к чужим ассетам;
механики мира.
```

Внешние материалы, добавленные в старый мир для удобства игры, не переносятся.

## Принцип пересборки

Все материалы должны пересобираться заново в структуре модуля:

```text
content/regions/
content/cities/
content/factions/
content/npcs/
content/monsters/
content/items/
content/rituals/
content/spirits/
content/handouts/
content/rolltables/
content/scenes/
```

Готовые старые базы Foundry не копируются напрямую. Каждый город, NPC, монстр, предмет или ритуал должен быть перенесён в чистую карточку по актуальному шаблону.

## Правило удобства для мастера

Контент должен раскладываться так, чтобы во время игры его было удобно искать.

Основное разделение:

```text
по Осколку;
по региону;
по городу;
по фракции;
по типу сущности;
по связи с конкретной сценой.
```

Если NPC относится к конкретному городу, он должен быть связан с этим городом. Если монстр относится к конкретному региону, он должен лежать в региональном разделе бестиария. Если фракция действует в нескольких регионах, это должно быть указано в карточке фракции.

## Итоговое правило

Новые документы — основа.

Старый мир — только справочник и источник отдельных точных интерпретаций.

Окрам и аналогичные исключения можно брать из старого мира, если старая версия точнее.

Compendium packs старого мира, внешний контент, музыка, пользователи, история игры и технические данные не переносятся.
