# Changelog

## 0.3.21

Срочное исправление безопасного обновления и боёвки:

- добавлен `scripts/aeriya-world-protection.js`: защитный слой Foundry блокирует удаление сцен, папок сцен и Actor Аэрии, а также вырезает из обновлений сцен пустые `tokens`, `walls`, `tiles`, `notes`, `lights`, `sounds`, `drawings`, чтобы импортёр больше не стирал пользовательскую работу на сценах;
- добавлен `scripts/aeriya-safe-content-guard.js`: ручные и автоматические импорты по умолчанию переводятся в безопасный режим `overwrite: false`, `deleteLegacyScenes: false`; разрушительная перезапись возможна только при явном `allowDestructiveOverwrite: true`;
- `scripts/aeriya-external-bestiary.js` обновлён до `external-bestiary-v4-descriptive-safe`: автоимпорт внешнего бестиария больше не перезаписывает существующих Actor по умолчанию, а новые Actor получают нормальные описания, среду, признаки, тактику и применение в кампании вместо заглушки;
- добавлен `scripts/aeriya-combat-tools.js`: создаётся макрос `Аэрия: Реакция в бою`, добавлен API `game.aeriya.combat`, восстановление реакции в начале хода, ручное восстановление, трата реакции и бросок восстановления 1к10;
- версия модуля поднята до `0.3.21`.

## 0.3.16

Исправлено окончательно по сценам:

- `scripts/aeriya-book-scenes.js` обновлён до `book-scenes-v4`;
- все 54 сцены из книги `Города и поселения Аэрии` теперь лежат прямо в папках регионов, без дополнительного вложения `Города / Поселения / Локации`;
- `Срединные Земли`, `Теневой Осколок / Пепельная Степь`, `Теневой Осколок / Лес Чёрного Шёпота`, `Теневой Осколок / Морозный Предел`, `Палящий Осколок / Выжженные Пустоши`, `Палящий Осколок / Подземный Предел`, `Палящий Осколок / Соляные Пустыни` теперь получают сцены напрямую в папку региона;
- добавлены обзорные служебные сцены для корневых папок и регионов: `Срединные Земли`, `Теневой Осколок`, `Пепельная Степь`, `Лес Чёрного Шёпота`, `Морозный Предел`, `Палящий Осколок`, `Выжженные Пустоши`, `Огненные Каньоны`, `Подземный Предел`, `Соляные Пустыни`;
- `game.aeriya.repairBookScenes()` теперь восстанавливает не только книжные сцены, но и обзорные сцены, чтобы в интерфейсе Foundry не было пустых региональных папок.

## 0.3.15

Исправлено:

- `scripts/aeriya-actor-ability-repair.js` обновлён до `actor-ability-repair-v2`;
- ремонт характеристик теперь затрагивает старые Actor из legacy-папок `Бестиарий`, включая ветки вроде `Бестиарий / Общие / Общие`, а не только `Aeria Core`;
- добавлен авто-ремонт при открытии листа NPC Actor: если у Aeria/бестиарного NPC все характеристики остались 10/10/10/10/10/10, модуль сразу пересчитывает и обновляет `system.abilities`;
- добавлен ручной метод `game.aeriya.repairSelectedActorAbilities({ overwrite: false })` для выбранных токенов;
- общий ремонт теперь запускается через 4 секунды после старта GM, а не через 15 секунд.

## 0.3.14

Исправлено:

- `scripts/aeriya-book-scenes.js` обновлён до `book-scenes-v3`;
- книжные сцены из `Города и поселения Аэрии` теперь проверяются и восстанавливаются при каждом старте GM, а не пропускаются из-за ранее записанной версии в world settings;

## 0.3.13

Исправлено по результатам повторной проверки в Foundry:

- добавлен `scripts/aeriya-actor-ability-repair.js` — ремонт Actor, у которых после старого импорта остались дефолтные характеристики 10/10/10/10/10/10;
- ремонт затрагивает только Aeria NPC Actor и пересчитывает характеристики по КО, типу существа, размеру, роли, названию и описанию;
- добавлен ручной метод `game.aeriya.repairActorAbilities({ overwrite: false })`;
- `scripts/aeriya-scene-cleanup.js` обновлён до `book-scene-cleanup-v2`;
- очистка сцен теперь удаляет не только пустые папки внутри `Aeria Core / Сцены`, но и legacy-папки `Пепельная степь`, `Огненные Каньоны`, `Ледяной Предел`, `Восток`, `Запад`, `Подземные`, если в них лежат старые не-книжные сцены;
- очистка сцен теперь запускается при каждом старте GM и после `game.aeriya.importAll(...)`, даже если версия установки уже была записана ранее.
