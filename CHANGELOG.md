# Changelog

## 0.3.10

Исправлено по результатам проверки в Foundry:

- `scripts/aeriya-actor-normalizer.js` обновлён до `actor-normalizer-v2`: нормализатор теперь корректно читает строку характеристик из markdown-таблицы и не принимает строку `|---:|---:|...|` за значения;
- уже созданные Actor из `content/actor-statblocks/**` при запуске модуля получают правильные `system.abilities`, поэтому верхний блок листа больше не должен оставаться 10/+0 при наличии характеристик в статблоке;
- во внешнем бестиарии заменён битый `icons/svg/monster.svg` на штатный `icons/svg/mystery-man.svg`, чтобы убрать 404 в консоли Foundry;
- внешний бестиарий обновлён до `external-bestiary-v3`, чтобы ранее созданные Actor получили исправленную иконку и prototype token;
- добавлен `scripts/aeriya-scene-cleanup.js`: очистка старых Aeria-сцен, которых нет в книге `Города и поселения Аэрии`, и рекурсивное удаление пустых legacy-папок внутри `Aeria Core / Сцены`;
- добавлен ручной метод `game.aeriya.cleanupBookSceneStructure({ deleteLegacyScenes: true })`.

## 0.3.9

Добавлено:

- добавлен `scripts/aeriya-calendar.js` — календарь Аэрии для Foundry;
- календарная модель: 20 дней в месяце, 18 месяцев в году, 360 дней в году;
- названия месяцев и годов стилизованы через бестиарий: месяцы низших и средних существ, годы сильнейших существ;
- у GM создаются макросы `Аэрия: Календарь` и `Аэрия: Долгий отдых` в папке `Aeria Core / Макросы`;
- при долгом отдыхе через макрос Аэрии календарь сдвигается на 1 день и выводит в чат сообщение вида `Настал 20 день месяца Песчаной Змеи, год Повелителя Бурь`;
- добавлены слушатели возможных событий долгого отдыха dnd5e/Foundry как автоподхват штатной кнопки отдыха;
- добавлен API `game.aeriya.calendar.show()`, `game.aeriya.calendar.advance(...)`, `game.aeriya.calendar.reset(...)`, `game.aeriya.calendar.longRestSelectedTokens(...)`.

## 0.3.8

Добавлено:

- добавлен `scripts/aeriya-weather-macro.js` — макрос неестественной погоды для Foundry;
- у GM автоматически создаётся макрос `Аэрия: Неестественная погода` в папке `Aeria Core / Макросы`;
- макрос бросает сезон, погодное явление, срок смены погоды и модификатор проклятия по таблицам из документа `Генератор Неестественной Погоды`;
- результат выводится в чат как карточка с описанием погоды, сроком действия и проклятием;
- добавлены ручные методы `game.aeriya.rollUnnaturalWeather(...)` и `game.aeriya.ensureWeatherMacro()`.

## 0.3.7

Добавлено:

- добавлен `scripts/aeriya-external-bestiary.js` — отдельный импортёр внешнего бестиария;
- в импорт внешнего бестиария включены 164 существа из загруженных документов: 45 из `Бестиарий ЭНОА, версия 8`, 20 из `Бестиарий мира Ведьмак для DnD 5`, 99 из `Тотальное убийство партии. Бестиарий часть 1`;
- существа создаются как полноценные `npc` Actor в папках `Aeria Core / Бестиарий / Внешние источники / ЭНОА`, `Ведьмак`, `ТПК`;
- для каждого Actor структурно заполняются КО, КД, хиты, скорость, характеристики, тип, размер, биография, prototype token и embedded Items: роль существа, мультиатака, основная атака, особое действие, а также титанические и легендарные метки, если они есть в источнике;
- `game.aeriya.importAll(...)` теперь дополнительно запускает импорт внешнего бестиария;
- добавлен ручной метод `game.aeriya.importExternalBestiary({ overwrite: true })`.

## 0.3.6

Исправлено по результатам аудита чарников и импорта:

- добавлен `scripts/aeriya-actor-normalizer.js` — безопасный пост-импортный нормализатор Actor-листов;
- после импорта НИПов и существ нормализатор перечитывает исходный статблок и дозаполняет структурные поля чарника: движение, чувства, языки, сопротивления, уязвимости, иммунитеты к урону и иммунитеты к состояниям;
- сырые строки `Спасброски`, `Навыки`, `Сопротивления`, `Уязвимости`, `Иммунитеты`, `Чувства`, `Языки` сохраняются во флагах Actor, чтобы мастер видел исходные значения даже там, где dnd5e-система не даёт безопасно разложить поле автоматически;
- `game.aeriya.importAll(...)` теперь после основного импорта дополнительно запускает нормализацию чарников;
- добавлен ручной метод `game.aeriya.normalizeActors({ overwrite: true })`;
- `content/generated/journal-content.json` больше не является пустым файлом и содержит валидный JSON-массив, чтобы импорт и `import:validate` не падали на разборе JSON.

## 0.3.5

Исправлено по результатам аудита:

- импорт сцен из книги `Города и поселения Аэрии` теперь сопоставляет старые и новые сцены по нормализованному имени, чтобы не плодить дубли из-за апострофов, кавычек, дефисов, регистра и вариантов `й/и`;
- ручной `game.aeriya.importAll({ overwrite: true })` теперь оборачивается книжным импортёром и после общего импорта снова проверяет/восстанавливает все сцены книги;
- книжный импортёр обновлён до схемы `book-scenes-v2`, чтобы Foundry повторно прогнал раскладку сцен по папкам.

## 0.3.4

Исправлено:

- добавлен отдельный импортёр сцен `scripts/aeriya-book-scenes.js` для полного реестра из книги `Города и поселения Аэрии`;
- все строки книги создаются как show-only Scene: города, поселения, форпосты, лагеря, станции, стоянки, руины и ключевые локации;
- сцены раскладываются по дереву книги: `Aeria Core / Сцены / Осколок / Регион / Города|Поселения|Локации`;
- при совпадении названия импортёр обновляет существующую сцену и переносит её в правильную папку, а не создаёт дубль;
- вручную назначенный фон сцены не затирается;
- добавлен ручной метод `game.aeriya.importBookScenes({ overwrite: true })`.

## 0.3.3

Исправлено:

- автоимпорт Foundry теперь использует новую схему `world-import-v4`, чтобы обновление повторно прошло в уже созданном мире;
- при `overwrite=true` у Journal Entry удаляются лишние старые страницы, если новая версия материала содержит меньше страниц;
- при `overwrite=true` у Actor полностью пересобираются embedded Items, включая очистку старых действий, если новая версия статблока больше не создаёт предметы;
- сохранены вручную назначенные изображения Actor и Prototype Token, если в манифесте ещё нет импортированного токена;
- исправлен разбор секций статблоков без рискованной динамической регулярки;
- сохранена конвертация `к` → `d` в формулах ХП и урона.

## 0.1.0

Начальная сборка каркаса Foundry-модуля Аэрии.

Добавлено:

- `module.json` с базовой конфигурацией модуля;
- стартовый ES-module `scripts/aeriya-core.js`;
- русская локализация;
- базовые стили;
- шаблон будущей панели мастера;
- правила игнорирования локальных миров, ассетов, аудио и служебных файлов Foundry.

Не добавлялись:

- аудиофайлы;
- пользовательские данные мира;
- fog of war;
- история чата;
- служебные LevelDB-файлы;
- изображения, карты и токены.
